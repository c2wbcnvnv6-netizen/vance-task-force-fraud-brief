# Federal Risk Explorer & Transparency Hub - Website Orchestrator Plan

## Project Name
Task Force Fraud Research → Federal Risk Explorer & Government Transparency Hub

## Primary Goals
- Transform from single static research brief into a professional, all-in-one public service knowledge hub.
- Focus on empowering users to investigate federal spending (especially HHS/Medicaid/Medicare) with risk signals.
- Maintain factual, empowering, public-service tone.
- Excellent UX: fast, mobile-friendly, accessible (WCAG), good contrast.

## Target Architecture (Vercel Static)
- Keep it lightweight and fast (no heavy framework if possible).
- Single-page app with smooth section navigation + dedicated sub-pages for major tools (FOIA Toolkit, Explorer, etc.).
- Or light multi-file structure (index.html + /explorer/ + /foia-toolkit/ + /tools/ etc.).
- Leverage existing Tailwind + Font Awesome + Chart.js.
- Add Leaflet (already in explorer) + lightweight JS for interactivity.

## Key Sections / Pages

1. **Homepage (revamped index.html)**
   - Hero: “Knowledge is Power” + strong value prop.
   - Clean top nav (Home | FOIA Toolkit | Federal Risk Explorer | Research Library | Tools | Submit a Tip | About).
   - Featured sections:
     - Federal Risk Explorer highlight
     - Quick FOIA Starter
     - Latest Research Briefs
     - Key Tools
   - Site-wide search bar (client-side or simple).
   - Strong CTAs.

2. **Federal Risk Explorer** (enhance /explorer/federal-risk-explorer.html)
   - Keep + improve interactive US map (HHS focus).
   - Powerful searchable/filterable data table below the map.
   - Advanced filters + export (CSV/JSON/PDF).
   - FOIA guidance for missing data.
   - Prominent “Submit a Tip” form.
   - Supporting charts/stats.

3. **FOIA Toolkit** (new dedicated page/section)
   - Interactive form builder.
   - Ready templates.
   - State + federal guidance.

4. **Research Library** 
   - Organized briefs and reports.

5. **Tools**
   - Integration points for document analysis, recipient tracing, etc.

6. **Submit a Tip**
   - Simple, private-friendly form.

## Design System
- Clean, modern, high-contrast (dark/light friendly).
- Professional government-transparency aesthetic.
- Consistent components (cards, buttons, tables, modals).
- Mobile-first.

## Data & Integration
- Keep explorer data driven by USAspending where possible.
- Clear fallbacks + FOIA guidance where data is limited.

## Deployment
- Use existing Vercel + GitHub connection.
- Use github-site-manager + vercel-integrator + deployment-autopilot patterns.

## Phased Approach (for marketable progress)
Phase 1: Homepage revamp + nav + basic explorer improvements + FOIA starter.
Phase 2: Full explorer table/filters + tip form.
Phase 3: Dedicated FOIA Toolkit + other pages + search.

## Success Criteria
- User can land on homepage and immediately understand value.
- Can navigate to powerful Risk Explorer with real utility.
- Can easily start a FOIA or submit a tip.
- Site feels trustworthy and professional.
